0 Week at a glance

raw http → Express → middleware → routes → views. The framework that runs most of the Node web.
  • Express.js — a thin, fast web framework for Node. Routing + middleware + views, minus the boilerplate.
  • Middleware — functions a request flows through, one after another. The heart of Express.
  • Routing — app.METHOD(path, handler): match a URL + method, run a handler.
  • Route params — capture pieces of the URL: /users/:id → req.params.id.
  • Template engine (Pug) — fill an HTML template with data on the server, send finished HTML.

New this week: the Express emulator. Examples marked 🚂 express emulator run a mini-Express right in this page — require('express'), real middleware, routing and res.render(). Hit your servers with the built-in REST tester (choose GET / POST / PUT / DELETE and send a body). Everything is editable.

1 Why Express?

Key terms framework middleware routing views convenience methods

Express.js is a Node.js web application framework — a robust set of features for building web apps and APIs, layered on top of the http module you met in Week 10.

Raw http gives you a hammer. Express gives you the whole workshop — but it's still just Node underneath.

The four things Express hands you out of the box:

  • Middleware — let a request flow through multiple functions before you answer it.
  • Routing — handle a request at a specific URL + method, cleanly.
  • Convenience methods — res.send(), res.json(), res.redirect()… no manual headers.
  • Views — dynamically render HTML with a template engine like Pug.

Same server, two ways

Both servers answer "Hello World" at /. Count the moving parts.
Raw http (Week 10)
http-way.js
const http = require('http');

const server = http.createServer((req, res) => {
  if (req.url === '/' && req.method === 'GET') {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Hello World!');
  } else {
    res.writeHead(404);
    res.end('Not found');
  }
});

server.listen(3000);
Express way
express-way.js
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(3000);

Express handles the method check, the status code, the Content-Type header and the 404 fallback for you. You describe what to answer, not how to wire it.

2 Hello World server — line by line

Key terms require('express') express() app.get app.listen process.env.PORT
Run it, then visit your server in the mini-browser. Change the message and run again!
hello-express.js
const express = require('express');   // import the module
const app = express();                // create an app instance

const port = process.env.PORT || 3000;   // pick a port

// When a GET request hits "/", run this callback.
// It gets the request (req) and the response (res).
app.get('/', function (req, res) {
  res.send('Hello World!');
});

// Start listening. The callback fires once the server is up.
app.listen(port, function () {
  console.log('Example app listening on port ' + port);
});
LineWhat it does
require('express')Loads the Express package (installed with npm install express).
express()Creates your application object, app — the thing you attach routes to.
app.get('/', cb)Registers a handler for GET requests to /. The callback receives req & res.
res.send(...)Sends the response — sets status 200 and the right headers automatically.
app.listen(port, cb)Opens the port and starts accepting requests.

3 Middleware — the heart of Express

Key terms (req, res, next) next() the stack order matters!

Middleware = a function that sits between the incoming request and the final response. It receives (req, res, next), can inspect or modify req/res, and then calls next() to pass control to the next function in line.

A request is a traveller at the airport. Each middleware is a checkpoint. next() waves them through to the next one.

Real middleware, running live

Run it, then visit any path. Watch the console — every request passes the logger first.
middleware.js
const express = require('express');
const app = express();

// A middleware: runs on EVERY request, then hands control on with next()
app.use(function (req, res, next) {
  console.log('[' + req.method + '] ' + req.url + ' — logged by middleware');
  req.visitedAt = new Date().toLocaleTimeString();   // add data for later
  next();   // ⬅️ forget this and the request hangs forever!
});

app.get('/', (req, res) => {
  res.send('Home page — you arrived at ' + req.visitedAt);
});

app.get('/about', (req, res) => {
  res.send('About page — logged at ' + req.visitedAt);
});

app.listen(3000);

The #1 Express bug: forgetting next(). Without it, the request stops at that middleware and the browser waits forever — just like forgetting res.end() in Week 10. Delete next() above and visit a page to feel it.

The two built-ins you'll use constantly

  • express.json() — parses incoming JSON request bodies into req.body.
  • express.static('public') — serves static files (HTML, CSS, images) from a folder.
Run it, then in the REST tester switch the method to POST, send a JSON body, and watch req.body come alive.
json-body.js
const express = require('express');
const app = express();

app.use(express.json());   // ⬅️ built-in: fills req.body from JSON

app.post('/echo', (req, res) => {
  // Without express.json(), req.body would be undefined!
  res.json({
    youSent: req.body,
    itemCount: Object.keys(req.body).length
  });
});

app.get('/', (req, res) => {
  res.send('POST some JSON to /echo — use the method selector below ↓');
});

app.listen(3000);

5 Routing

Key terms app.METHOD(path, handler) get · post · put · delete app.all

A route is a URL the server listens to and responds to when a client makes a request. The shape is always app.METHOD(PATH, HANDLER).

  • METHOD — get, post, put, delete, all, and more (lower-case).
  • PATH — a string like '/users' (or a pattern — see next section).
  • HANDLER — the callback (req, res) => { ... } that answers.
routing.js · visit /, /users, /contact
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('<h1>Welcome home</h1><a href="/users">See users</a>');
});

app.get('/users', (req, res) => {
  res.send('List of users: Ada, Linus, Grace');
});

app.get('/contact', (req, res) => {
  res.send('Email us at hello@archipelago.dev');
});

app.listen(3000, () => console.log('Server running on port 3000'));

6 Route paths

Key terms exact strings patterns ? + * ( )
Paths can be plain strings or patterns using ? + * ( ).
RouteMatches
app.get('/', ...)The root: /
app.get('/about', ...)Exactly /about
app.get('/ab(cd)?e', ...)/abe and /abcde — the (cd) is optional
app.get('/ab*cd', ...)/abcd, /abXYZcd — * matches anything
route-paths.js · try /abe and /abcde
const express = require('express');
const app = express();

app.get('/', (req, res) => res.send('root'));

app.get('/about', (req, res) => res.send('about'));

// (cd) is optional → matches /abe AND /abcde
app.get('/ab(cd)?e', (req, res) => res.send('matched the ab(cd)?e pattern!'));

app.listen(3000);

7 Route parameters

Key terms :name req.params named segments

Route parameters are named URL segments that capture values at a position in the URL. Express stores them in the req.params object, keyed by name.

Path /users/:userId/books/:bookId · URL /users/34/books/8989 · req.params = { userId: "34", bookId: "8989" }

The matcher — type a pattern and a URL

Live — a server that reads params

Run it, then visit /users/34/books/8989 — or change the numbers.
params.js
const express = require('express');
const app = express();

app.get('/users/:userId/books/:bookId', function (req, res) {
  // Everything after the : is captured into req.params
  res.json(req.params);
});

app.get('/', (req, res) =>
  res.send('Try visiting /users/34/books/8989 in the tester below ↓'));

app.listen(3000);

8 The Request & Response objects

Read from req req.params req.query req.body req.ip
req = everything the client sent. res = everything you send back.
req propertyHolds
req.paramsNamed route parameters — /users/:id → req.params.id.
req.queryURL query-string values — ?sort=asc → req.query.sort.
req.bodyParsed request body — needs express.json() (or a body-parser).
req.ipThe client's IP address.
req.method / req.pathThe HTTP verb and the path portion of the URL.
res methodSends
res.send(body)Text or HTML — sets the content type automatically.
res.json(obj)A JSON response — the workhorse of APIs.
res.status(code)Sets the status code — chainable: res.status(404).send('Nope').
res.redirect(url)Sends a redirect to another URL.
res.render(view, data)Renders a template (Pug) with data into HTML.
Run it, then visit /search?q=otter&sort=asc to see req.query in action.
req-res.js
const express = require('express');
const app = express();

app.get('/search', (req, res) => {
  // /search?q=otter&sort=asc  →  req.query = { q: 'otter', sort: 'asc' }
  const term = req.query.q || '(nothing)';
  const sort = req.query.sort || 'default';

  if (!req.query.q) {
    return res.status(400).json({ error: 'add ?q=your-search to the URL' });
  }

  res.json({
    youSearchedFor: term,
    sortedBy: sort,
    from: req.ip,
    method: req.method
  });
});

app.get('/', (req, res) => res.send('Try /search?q=otter&sort=asc'));

app.listen(3000);

9 HTTP methods & REST

The verbs GET — read POST — create PUT — update DELETE — remove
Same URL, different verb, different action. That's REST: GET read · POST create · PUT update · DELETE remove.

The fun one — a real REST API you can poke

This is a complete in-memory albums API. Run it, then use the REST tester's method selector to GET the list, POST a new album (send a JSON body!), and DELETE one by id. The data really changes between requests.

albums-api.js · a mini REST server
const express = require('express');
const app = express();
app.use(express.json());

// Our "database" — just an array (Model, Week 10 style)
let albums = [
  { id: 1, title: "Await for It", artist: "Async Attitude" },
  { id: 2, title: "Event Loop Grooves", artist: "The Callbacks" }
];
let nextId = 3;

// READ all  →  GET /albums
app.get('/albums', (req, res) => res.json(albums));

// READ one  →  GET /albums/2
app.get('/albums/:id', (req, res) => {
  const album = albums.find(a => a.id === Number(req.params.id));
  if (!album) return res.status(404).json({ error: 'no album with id ' + req.params.id });
  res.json(album);
});

// CREATE  →  POST /albums   body: { "title": "...", "artist": "..." }
app.post('/albums', (req, res) => {
  const album = { id: nextId++, title: req.body.title, artist: req.body.artist };
  albums.push(album);
  res.status(201).json(album);
});

// DELETE  →  DELETE /albums/1
app.delete('/albums/:id', (req, res) => {
  albums = albums.filter(a => a.id !== Number(req.params.id));
  res.json({ deleted: req.params.id, remaining: albums.length });
});

app.listen(3000, () => console.log('Albums API ready — try GET /albums'));

Class challenge: after running, POST { "title": "My Album", "artist": "Me" } to /albums, then GET /albums and watch it appear. Then DELETE /albums/1 and GET again. Add a PUT /albums/:id route yourself!

10 Serving static files

Key terms express.static('public') no route needed HTML · CSS · images
Frontend files (HTML/CSS/images) don't need a route each — point Express at a folder and it serves them all.
static.js
const express = require('express');
const app = express();

// Anything inside the /public folder is served automatically:
//   public/index.html   →  http://localhost:3000/index.html
//   public/style.css    →  http://localhost:3000/style.css
//   public/logo.png     →  http://localhost:3000/logo.png
app.use(express.static('public'));

app.listen(3000);
This is how frontend + backend live together: static middleware serves the frontend, routes serve the data.

11 Template engines & Pug

Key terms template engine Pug res.render() #{variables}

A template engine lets you write an HTML template with placeholders, then fill them with real data in JavaScript. Node replaces the variables and sends finished HTML to the client.

Server gets GET /dog → fills a template with { word: "dog", definition: "…" } → returns a complete page.
  • Popular engines: Pug, EJS, Handlebars.
  • This course uses Pug — cleaner, indentation-based, no closing tags.

Setting up Pug in Express

app.js
// 1. install:  npm install pug
const express = require('express');
const app = express();

// 2. tell Express to use Pug, and where the templates live
app.set('view engine', 'pug');
app.set('views', './views');

// 3. create views/index.pug, then render it with data:
app.get('/', (req, res) => {
  res.render('index', { name: 'John', items: ['Apple', 'Banana', 'Cherry'] });
});

app.listen(3000);

13 MVC in Express

                    

                        Key terms
                        Model = data
                        View = Pug
                        Controller = routes
                    

                    

                        Week 10 taught MVC. Express makes it natural: Model = data · Controller = your route handlers · View = Pug.
                    

                    
A complete little MVC app — a Model (array), a Controller (the route) and a View (rendered Pug). Run it and visit /:

                    

                        
mvc-app.js

                        
const express = require('express');
const app = express();
app.set('view engine', 'pug');

// 🧠 MODEL — the data + logic (no database yet, just an array)
const students = [
  { name: 'Ada Lovelace', grade: 'A+' },
  { name: 'Linus Torvalds', grade: 'A' },
  { name: 'Grace Hopper', grade: 'A+' }
];

// 🚦 CONTROLLER — the route decides what to do and picks a view
app.get('/', (req, res) => {
  res.render('students', { title: 'Class List', students: students });

app.listen(3000);

15 Self-check quiz

                    
No grades, instant feedback. Can you get all eight?


                    

                        
Q1. What is Express.js?

                        
                        
                        
                        
Express is a web application framework for Node.js — it sits on top of the http module and gives you routing, middleware and view rendering.

                    


                    

                        
Q2. A middleware function runs but never calls next(). What happens?

                        
                        
                        
                        
next() passes control to the next function in the stack. Skip it (and don't send a response) and the request hangs — the classic Express bug.

                    


                    

                        
Q3. Which line registers a handler for POST requests to /albums?

                        
                        
                        
                        
The shape is app.METHOD(path, handler) — the method is the function name: app.post(...), app.get(...), etc.

                    


                    

                        
Q4. Route /users/:userId receives a request for /users/42. Where is "42"?

                        
                        
                        
                        
Named route segments (the :userId part) are captured into req.params. Query strings go to req.query; JSON bodies to req.body.

                    


                    

                        
Q5. To read a JSON request body from req.body, you must first…

                        
                        
                        
                        
Without express.json() parsing the body, req.body is undefined. It's a built-in middleware you register once with app.use().

                    


                    

                        
Q6. In REST, which verb creates a new resource?

                        
                        
                        
                        
GET reads, POST creates, PUT updates, DELETE removes. Same URL, different verb, different action.

                    


                    

                        
Q7. What does app.use(express.static('public')) do?

                        
                        
                        
                        
express.static is built-in middleware that serves static assets (HTML, CSS, images) straight from a folder — public/style.css becomes /style.css.

                    


                    

                        
Q8. In Express + Pug MVC, which part is the View?

                        
                        
                        
                        
Model = the data/array · Controller = the route handler that decides & calls res.render · View = the Pug template that presents the data.
